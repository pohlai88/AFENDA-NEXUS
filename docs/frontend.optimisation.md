## Plan: App Shell — Enterprise-Grade Overhaul

**TL;DR:** The shell has structural debt: legacy dead code, duplicate icon maps across 3 files, stale command palette data, raw HTML bypassing shadcn, missing shell-level error boundary, breadcrumbs using `<a>` instead of `<Link>`, and a `document.dispatchEvent` hack for search. This plan addresses **13 improvements** across 8 files — cleaning up ~150 lines of dead code, consolidating 3 icon maps into 1, upgrading the command palette to use the live module system, extracting the header into its own component, and adding the missing shell error boundary.

**Steps**

1. **Create shared icon registry** — New file apps/web/src/lib/modules/icon-map.ts
   - Consolidate the 3 duplicate `iconMap` declarations (from sidebar-nav.tsx ~50 icons, module-sidebar.tsx ~7 icons, command-palette.tsx ~14 icons) into a single canonical export: `export const ICON_MAP: Record<string, LucideIcon>`
   - Export a helper `getIcon(name: string, fallback?: LucideIcon): LucideIcon`
   - All 3 consumers switch to importing from this module instead of maintaining their own maps

2. **Remove legacy dead code from sidebar-nav.tsx**
   - Delete `SidebarNav` function (lines ~349–380) — confirmed **zero external imports**; only `DomainNav` is consumed (by domain-sidebar.tsx)
   - Delete `FinanceSidebarNav` function (lines ~189–205) — only called by the deleted `SidebarNav`
   - Delete `LegacyNavGroup` function (lines ~382–420) — only called by the deleted `SidebarNav`
   - Remove the `import { navigationConfig } from '@/lib/constants'` (line 347) — no longer needed
   - Replace inline `iconMap` + `NavIcon` with import from the new shared icon registry
   - Net: **~100 lines removed**, file drops from 420 → ~250 lines

3. **Extract topbar into shell-header.tsx**
   - Extract the inline `<header>` block (lines 80–105 of app-shell.tsx) into a dedicated `ShellHeader` component
   - Move `RightSidebarTrigger` into the same file (it's an internal helper of the header)
   - Props: `user`, `logoutAction`, `onOpenCommandPalette`
   - This separates concerns and makes the shell orchestrator cleaner

4. **Fix search button → proper state management**
   - Replace the `document.dispatchEvent(new KeyboardEvent(...))` hack in app-shell.tsx (line 92) with React state:
   - Lift `commandPaletteOpen` state to `AppShell`, pass `open`/`onOpenChange` props to `CommandPalette`
   - The search `<Button>` simply calls `setCommandPaletteOpen(true)` — no DOM event simulation
   - `CommandPalette` receives `open`/`onOpenChange` props alongside its own keyboard listener

5. **Fix CommandPalette DOM position**
   - Move `<CommandPalette />` out of `<header>` and render it as a sibling at the **root level** of the shell (inside `TenantProvider` but outside the sidebar/header tree)
   - `CommandDialog` uses a portal internally, but semantic placement should be at root

6. **Upgrade CommandPalette to use module system** — command-palette.tsx

   - This means the command palette shows the **same navigation** as the sidebar (including admin items when visible), not a stale legacy list
   - Remove the local `iconMap` — use shared registry from Step 1
   - Remove the `import { navigationConfig }` line

**Decisions**
- **Keep custom breadcrumb over installing shadcn Breadcrumb**: The custom implementation in `PageHeader` is well-tested, accessible (`aria-label`, `aria-current`), and widely consumed. Installing shadcn's `Breadcrumb` component would require touching 20+ pages for marginal benefit. The critical fix is `<a>` → `<Link>`.
- **Notification bell deferred**: Adding a notification system (bell icon + feed in topbar) is a feature, not a code-quality fix. It requires backend infrastructure (notification service, real-time channel). Flagged for future sprint.
- **`navigationConfig` deprecation over deletion**: Since it has a test file, deleting it outright would break tests. Marking `@deprecated` now and removing in a follow-up is safer.
- **Icon map as standalone module (not in module-spec)**: The icon map serves UI rendering and should live in `lib/modules/icon-map.ts`, not pollute the data-only `module-spec.ts`.
